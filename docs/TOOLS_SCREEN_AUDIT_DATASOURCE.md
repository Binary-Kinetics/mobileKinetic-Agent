# Tools Screen Audit Data Source

**Status**: Data structure specification for mK:a Tools screen integration
**Date**: 2026-02-25
**Purpose**: Define how endpoint audit data feeds the Tools screen UI and status system

---

## Overview

The Tools Screen displays all available device capabilities in a unified interface. Rather than hard-coding endpoints, the screen should consume audit data that tracks endpoint status, permissions, parameters, and constraints. This document specifies:

1. The source data structure (endpoint audit format)
2. How the audit data maps to UI status indicators
3. The pipeline for seeding and updating tool status
4. Integration with on-device audit generation and validation

---

## Part 1: Endpoint Audit Structure

### Source File Location
- **On-device**: `~/Memory/endpoint_audit_YYYY-MM-DD.md` (generated by on-device Claude)
- **Format**: Structured markdown with categorized endpoint listings
- **Parsing**: Either regex-based (for markdown) or convert to JSON for machine consumption
- **Update frequency**: Periodic (after new DeviceApiServer.kt endpoints added, or on user request)

Endpoints work but have inherent platform constraints that cannot be worked around.

**Format in audit**:
```

  - Impact: Location will be null or stale indoors

  - Impact: Prefetch may fail on restricted sites
  - Workaround: Manual browser open instead of prefetch
```

**Quick Actions**:
- "Learn More" link to explanation
- "Request Alternative" button to suggest workarounds

## Part 4: Seeding and Update Pipeline

### Initial Seed (First Launch or After Build)

1. **On-Device Claude triggers audit**:
   - Queries DeviceApiServer.kt for all registered endpoints
   - Executes test requests for each endpoint
   - Categorizes based on response and error handling
   - Generates `endpoint_audit_YYYY-MM-DD.md` in `~/Memory/`

2. **App reads audit on startup**:
   ```kotlin
   // In MainViewModel or Tools screen initialization
   val auditFile = "~/Memory/endpoint_audit_*.md"
   if (auditFile.exists()) {
       parseAuditMarkdown(auditFile)
       seedToolDatabase()
   } else {
       useHardcodedDefaults()  // Fallback
   }
   ```

3. **Parse audit and insert into Room DB**:
   - Regex to extract endpoint blocks
   - Map each endpoint to Tool entity
   - Store relationships (permissions, parameters)

### Ongoing Updates

1. **When new endpoints added to DeviceApiServer.kt**:
   - Developer adds endpoint to code
   - Includes test case in unit tests
   - On-device Claude is prompted to run audit
   - Audit includes new endpoint in next scheduled run

2. **Periodic audit refresh** (weekly or on user request):
   - User can tap "Refresh Tool Status" in Tools screen
   - Triggers on-device audit script
   - New `endpoint_audit_YYYY-MM-DD.md` overwrites old
   - App reads new file and updates Room DB

3. **Audit auto-triggers**:
   - After app update (to validate new endpoints)
   - If a tool test fails (to catch regressions)
   - On 7-day interval (configurable)

## Part 5: Audit File Parsing Strategy

### Option A: Markdown Parsing (Current Format)

**Pros**: Human-readable, already being generated
**Cons**: Fragile to format changes, requires regex

**Parser pseudocode**:
```python
def parse_audit_markdown(file_path):
    with open(file_path) as f:
        content = f.read()

    tools = []

    # Extract each status section
    for section_match in re.finditer(r'^## (.*?)\s*\((\d+)\)', content, re.MULTILINE):
        status = section_match.group(1).strip()  # "GO - Read Endpoints"
        count = int(section_match.group(2))

        # Extract endpoints in this section
        section_text = content[section_match.end() : next_section_or_end]
        for endpoint_match in re.finditer(r'^- (\S+)\s*-\s*(.+)$', section_text, re.MULTILINE):
            endpoint_id = endpoint_match.group(1)  # "sms/list"
            description = endpoint_match.group(2)

            tools.append({
                'endpoint_id': endpoint_id,
                'status': map_section_to_status(status),
                'description': description,
                'category': infer_category(endpoint_id),
                'tier': infer_tier(endpoint_id)
            })

    return tools
```

## Part 7: On-Device Audit Generation (On-Device Claude Prompt)

### Audit Generation Script

The on-device Claude maintains and executes an audit script that:

1. Imports `DeviceApiServer` registry
2. Enumerates all registered endpoints
3. Tests each endpoint with reasonable defaults
4. Categorizes based on results
5. Writes `endpoint_audit_YYYY-MM-DD.md`

**File**: `~/scripts/audit_device_api.py`

```python
#!/usr/bin/env python3
"""
Audit all Device API endpoints on mK:a.
Generate categorized status report for Tools screen.
"""

import requests
import json
from datetime import datetime
from enum import Enum

class EndpointStatus(Enum):
    GO = "GO"
    NEEDS_PERMISSION = "NEEDS_PERMISSION"
    PARAMETER_ISSUE = "PARAMETER_ISSUES"
    BY_DESIGN = "BY_DESIGN"
    NOT_TESTED = "NOT_TESTED"
    BROKEN = "BROKEN"

def test_endpoint(endpoint_id, method, port=5563, params=None, body=None):
    """Test a single endpoint and return status."""
    url = f"http://localhost:{port}/{endpoint_id}"

    try:
        if method == "GET":
            response = requests.get(url, params=params, timeout=5)
        else:
            response = requests.post(url, json=body, timeout=5)

        if response.status_code == 200:
            return EndpointStatus.GO
        elif response.status_code == 403:
            return EndpointStatus.NEEDS_PERMISSION
        elif response.status_code == 400:
            return EndpointStatus.PARAMETER_ISSUE
        else:
            return EndpointStatus.BROKEN
    except requests.exceptions.Timeout:
        return EndpointStatus.NOT_TESTED
    except Exception as e:
        return EndpointStatus.BROKEN

def generate_audit():
    """Generate full endpoint audit."""

    endpoints = {
        "GO": {
            "Read": [
                ("sms/list", "List all SMS messages", "GET"),
                ("contacts", "List all contacts", "GET"),
                ("battery", "Get battery level, health, temperature", "GET"),
                # ... more endpoints ...
            ],
            "Write": [
                ("clipboard/set", "Write text to clipboard", "POST"),
                ("torch", "Toggle flashlight on/off", "POST"),
                # ... more endpoints ...
            ]
        },
        "NEEDS_PERMISSION": [
            ("phone/state", "Phone state (requires READ_PHONE_STATE)", "GET"),
            # ...
        ],
        # ... other categories ...
    }

    audit_date = datetime.now().strftime("%Y-%m-%d")
    output = f"# Device API Endpoint Audit - {audit_date}\n\n"

    # Test each endpoint and categorize
    results = {status.value: [] for status in EndpointStatus}

    for category, endpoints_list in endpoints.items():
        for endpoint_id, description, method in endpoints_list:
            status = test_endpoint(endpoint_id, method)
            results[status.value].append({
                'id': endpoint_id,
                'description': description,
                'method': method
            })

    # Write audit file
    audit_file = f"~/Memory/endpoint_audit_{audit_date}.md"
    with open(audit_file, 'w') as f:
        f.write(output)

        for status, endpoints_list in results.items():
            f.write(f"\n## {status} ({len(endpoints_list)})\n")
            for ep in endpoints_list:
                f.write(f"- {ep['id']} - {ep['description']}\n")

    print(f"Audit complete: {audit_file}")
    return audit_file

if __name__ == "__main__":
    generate_audit()
```
